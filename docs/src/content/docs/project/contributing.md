## Before you open a PR

1. Fork or branch from the latest `main`.
2. Run and validate the relevant apps locally.
3. Keep changes scoped and update docs when behavior changes.

## Useful commands

From repository root:

```bash
npm run dev:frontend
npm run build
npm run test
npm run lint
```

## Areas to contribute

- API route coverage and validation
- Collection and binder UX improvements
- Cache service reliability
- Mobile app feature parity
- Documentation and onboarding quality
